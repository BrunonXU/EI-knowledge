Terminology

先分清：状态表示、动作表示与执行接口

EEF 描述「手去哪」，Joints 描述「各关节怎么动」—— 两者不是维度之争，而是抽象层级之争

EEF / Task-space action

[Δx, Δy, Δz, Δroll, Δpitch, Δyaw, g] ∈ ℝ7

这里讨论的是末端 6-DoF 位姿增量 + 夹爪值；EEF frame 本身是坐标系，二者不要混用

Joints（关节空间）

q ∈ ℝn （G2 上半身 n = 14）

以智元 G2 为例：双臂各 7 关节，数据即关节角向量

FK（正运动学）

FK: q → EEF 位姿

确定性、唯一解

IK（逆运动学）

IK: EEF 位姿 → q

可能多解、需选择策略 —— 后文讨论的关键环节

Three Viewpoints

核心矛盾：任务语义通用性 vs 本体执行精度

01
Joint 保留本体运动先验 示范里的肘部偏好、连续 IK 分支与限位习惯都被保留下来；固定本体控制稳定，但跨本体迁移受限

Execution Authority

统一动作向量之后，机器人仍必须确定唯一执行接口

ât = [ âjointt, âEEFt, âgrippert, … ],　mt = active mask

PATH A · JOINT AUTHORITATIVE

Joint字段拥有控制权

Δq̂t:t+H → qt + Δq̂ → Joint Controller

  • 不需要在线IK
  • 保留示范构型与joint branch
  • 更适合固定本体

PATH B · EEF AUTHORITATIVE

EEF字段拥有控制权

ΔT̂EEF → Tt ⊕ ΔT̂ → IK / OSC / WBC → q / q̇ / τ

  • 任务几何更容易跨本体共享
  • 执行依赖IK、限位与奇异点处理
  • 还需碰撞与null-space策略

PATH C · JOINT + EEF CO-EXIST

两个字段存在，但只能有一个主执行

C1 · Joint主执行，EEF辅助约束

q̂ → Joint Controller；EEF用于FK(q̂)≈T̂EEF consistency loss、task-space monitoring。

C2 · EEF主执行，Joint提供构型偏好

T̂EEF → IK；q̂或current q作为IK seed、null-space posture target、joint-limit prior。

统一向量允许同时表达多个空间，但控制权必须明确；不能让两个独立控制器争夺同一组关节。

Deep Dive · Delta

Chunk-wise delta 让整段目标锚定同一观测帧，避免表示层的逐步漂移

01Step-wise逐步增量
at = qt − qt−1 → qt = qt−1 + at

每个标签参考上一个目标，预测偏差可能在目标重建时逐步传递 —— 像每一步都用上一落点重新量距离

02Chunk-wise锚定增量
at+k = qt+k − qt → qt+k = qt + at+k

整段锚定同一观测点 qt，标签彼此独立、天然并行 —— 每个目标都从同一地标量起

03为什么 Delta 优于 Absolute

Absolute：直接输出 qtarget，不同机器人绝对值分布差异巨大 → 需拟合全局关节空间，泛化差

Delta：输出 Δq = qtarget − qcurrent，分布紧凑（均值≈0、方差小）→ 与起始状态无关，泛化强

04EEF 的特殊性SE(3) 上的 delta 不是加法
Ttarget = ΔT · Tcurrent （位置可加，旋转是矩阵 / 四元数乘法）

⚠ EEF delta 需要群运算，工程实现比 joint delta 复杂 —— 这也是固定本体 Joint delta 更优的原因之一

边界：这里讨论的是 action chunk 的标签参数化；真实执行通常仍会闭环重规划，因此不等同于“整段开环执行必然不漂移”。

Two Different Problems

跨本体学习有两个不同的问题

GAP A · REPRESENTATION / TRANSFER

人类经验 → 可共享规律 → 机器人动作意图

差异：morphology、速度、grasp style、camera ego-motion

human experience ≠ robot execution

对应方法：relative EEF、canonical action、world supervision、EgoWAM

GAP B · EXECUTION

EEF / task intent → 本体约束 → feasible joints

约束：IK 多解、joint limit、collision、singularity、continuity、contact / balance

EEF intent → IK / OSC / WBC → q / q̇ / τ

对应方法：IK、OSC / WBC、IKSel、embodiment adapter

EgoWAM 处理“什么知识可以共享”；IKSel 处理“这台机器人怎样更快找到一个 IK 解”。

Controller Selection

我们应该如何选择合适的 IK / 控制器？

先确定控制层级与训练接口，再比较求解器；排行榜速度不是第一判断标准

01 · 策略到底输出什么？

Joint target → joint controller；EEF pose → IK / OSC；全身接触与平衡 → WBC / MPC。先选层级，不能拿一个 IK 包打天下。

02 · 数据由谁生成？

优先复用采集时的控制器、厂商验证接口或 benchmark 标准控制器，避免训练与部署的关节分布发生变化。

03 · 闭环真正关心什么？

连续性、joint limit / collision / singularity、求解失败恢复，以及目标控制频率下的稳定延迟。

IK Summary

三个核心认知

EEF 方案的上限，最终由运动学可达性与控制接口共同决定

一、EEF 到执行的必经桥梁

模型输出 EEF 后，仍需 IK / OSC / WBC 映射到关节或力矩；控制接口决定执行效果。

二、数据侧不可或缺

脚本采集、MimicGen 扩充、仿真生成（Isaac Sim）都依赖 IK 从 EEF 反算平滑关节轨迹。

三、选型与速度

多解歧义需合理种子策略（如 IKSel）；高频回路要求快且不牺牲解的质量。

IK 既决定执行质量，也是数据流水线的关键一环 —— 选对 IK、又快又稳，是 EEF 方案可用的最后一块拼图。

论文 3：IKSel

Inverse Kinematics Seed Selection for Fast IK Convergence · arxiv 2503.22234

核心：先建机器人的关节姿态 → 末端位姿运动学地图，再为 Numerical IK 选起点

① 采样关节空间在 joint limits 内均匀生成 qi
② 计算并标注xi=FK(qi)
同时保存 J(qi)+
③ 建 KDTree按 TCP pose xi 索引每个 qi
④ 在线查询输入 target x*
输出 seed q0
1

Workspace 先筛“近”：从地图中找一组 xi 接近 target x* 的候选；它们的手都在目标附近，但关节构型可能完全不同。

2

Jacobian 再预测“小”：对每个候选估计 Δqi=J(qi)+(x*−xi)；优先选择预计一步关节修正量 ‖Δqi‖ 最小的 seed。

3

Numerical IK 求解；失败后换“远”分支：若迭代撞到 joint limit / 局部极值，就在优质候选中改选关节构型差异更大的 seed，避开原来的 basin。

记住三个词：Workspace“近” → 预测 Δq“小” → 失败后 Joint Space“远”

IKSel 只选择更有希望收敛的初始关节构型 q0；Numerical IK 才计算最终解 q*。论文这里没有用 current q 做连续性排序。

它解决的问题

避免“末端最近但局部运动学很差”的 seed，让数值 IK 更快、更容易收敛。

Our Hypothesis · Not a Paper Conclusion

研究假设：共享任务表征 + 本体执行适配器

EgoWAM 验证 world supervision 能改善共享表征；下面的 EEF→Joint adapter 是我们的新增假设，尚需实验验证。

01 · DATA

Human + Robot Experience

Ego video、语言、手/EEF轨迹、robot state，以及对象/场景的未来变化。

→

02 · SHARED MODEL

Physics-aware Representation

学习“动作意图 → 对象/场景变化”，输出 anchored relative EEF intent 或 task-dynamics latent。

→

03 · EMBODIMENT ADAPTER

Feasible Joint Realization

输入 EEF intent + current q + URDF / limits / collision，输出 seed、posture、trajectory 或 feasibility score。

优先与控制器结合，而非宣称替代

adapter 可为 IK / WBC 预测 seed、null-space posture 或可行性分数；硬约束仍由控制器保证。

评估也要解耦

上层看OOD对象/场景与物理预测；下层看IK成功率、连续性、限位、碰撞与执行误差。

待验证：这种分层能否在不牺牲安全与约束保证的前提下，减少每台机器人重新学习执行映射的数据成本。

核心结论

01

固定本体上 Joint 更稳定；跨本体学习中 EEF 表示更容易共享任务几何

02

UMI、GEAR-VLA、GR00T N1.7 名字不同，本质都是锚定当前观测的 relative EEF chunk

03

GEAR-VLA 证明跨本体表示有效，但把 EEF→Joint 的问题留给 low-level interface

04

EgoWAM 改善 representation transfer，但没有解决 EEF→Joint 的 execution gap

05

研究假设：共享表征输出 EEF intent，本体 adapter 与 IK/WBC 协同实现可执行 joints

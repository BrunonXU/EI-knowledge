Terminology

基础定义：EEF 与 Joints

EEF (End-Effector Frame)

末端执行器的位姿，通常用 SE(3) 表示：

Tee ∈ SE(3) = (R, p)

其中 R ∈ SO(3) 为旋转矩阵，p ∈ ℝ³ 为位置向量。

Joints（关节空间）

机器人各关节角度构成的向量：

q ∈ ℝn

其中 n 为机器人自由度数（DoF）。

FK（正运动学）

FK: q → Tee

从关节角到末端位姿的映射，确定性、唯一解。

IK（逆运动学）

IK: Tee → q

从末端位姿反求关节角，可能多解、需选择策略。

核心关系：FK 是确定性的正向过程；IK 是其逆过程，存在多解歧义，是后文讨论的关键环节。

IK 的角色

IK 的角色 —— EEF 与 Joints 的桥接器

从 EEF 到机器人可执行的关节角度，需要经过 IK 求解；而 FK 则是正向过程，即已知 Joints 求解末端位姿。

若模型输出为 EEF，则需经由特定 IK 算法或控制器完成从 EEF 到 Joints 的映射。因此，最终执行效果不仅取决于模型 EEF 预测的精度，同时受限于所采用的 IK 算法的质量与有效性。

OpenVLA 架构：输出 7-DoF delta EEF action，明确声明需下游 IK/控制器映射到关节

各模型在 EEF 输出处的声明

  • OpenVLA：输出 delta EEF，需下游 IK/控制器映射到关节
  • OpenPI：同样输出 EEF action，执行依赖环境侧控制器
  • GR00T N1.7：提供 EEF + Joint 双 head，但最终执行取 Joint
  • LINGBot VLA 2.0：输出末端位姿，通过机器人自带控制接口执行

主题一

Joint vs EEF 的本质探讨

Three Viewpoints
1
Joint 照葫芦画瓢
  • Joint 离开了本体换一个机器人，本质照葫芦画瓢
  • 不是最优的终局之解，只是现在工程上表现力比较好
  • 数据绑定特定机器人构型，泛化能力受限
2
EEF 能学任务本意但忽略 IK 局限
  • 学习 EEF 能学到任务本意
  • 但忽略本体 IK 局限性
  • 胳膊如何过去很重要，不能只在乎末端执行器如何到达和执行
  • IK 多解歧义导致执行时抖动或失败
3
EE + 关节状态的融合方案
  • 使用 EE 提供共享任务几何
  • 同时保留关节状态或本体配置来保证可执行性
  • GEAR-VLA 即为此方向的代表实现
  • 兼顾跨本体迁移与单本体精度

核心矛盾：任务语义通用性 vs 本体执行精度 — 下面三篇论文分别从不同角度给出方案

实践路线

大家的做法 —— 两种路线

路线一：直接输出 Joint

直接输出关节角度，但输入隐含 EEF 信息

以 GR00T N1.7 DROID 配置为例，输入同时包含 relative EEF 与 relative joint，输出取 joint head 执行。

  • 输入容纳 EEF + Joint + Gripper 多字段
  • 推理执行时取 joint_position 输出
  • 从而避开在线 IK 过程

路线二：小模型隐含先验

用 encoder 对应的感知信息训练一个小模型

如 GR00T N1.7 的 SONIC / WBC 方案，通过训练一个轻量模型来隐含运动学和控制先验信息。

  • 不直接输出 EEF 或 Joint
  • 小模型学习感知到控制的映射
  • 隐含了运动学约束与控制先验

本质解释

路线一并非物理上"没有 IK"，而是模型从示范数据中隐式学习了本体的关节运动规律，在线时不再调用单独的 IK 求解器。EEF 信息在输入端提供几何理解，Joint 在输出端保证执行稳定性。

1EEF 提供通用任务几何
2Joint 保留目标本体的肩肘构型
3执行时绕过显式在线 IK
4可通过 FK consistency 约束两输出一致

论文支撑 1 — 深入分析

为什么 Delta 更易学习？

原始问题

图像 → 全局构型

Delta 转化

图像+状态 → 局部变化

局部变化特点：均值接近 0、方差小、分布集中、易于共享动作模式

Joint 更强的三大原因

1

避免 IK 误差

IK多解歧义即是误差来源

2

数据在关节流形上

示范轨迹包含固定肘部偏好、连续IK分支、避限位习惯

3

强生成模型可建模多模态分布

Flow Matching/Diffusion能表达多解，弱回归会平均化

⚠ 局限性
  • →EEF性能高度依赖IK和笛卡尔控制器质量
  • →跨本体实验规模小，未覆盖force/tactile/灵巧手
  • →未来可能不是二选一，而是根据任务阶段动态混合

结论

固定本体：Joint + chunk-wise delta 最优

跨本体迁移：EEF + chunk-wise delta 更适合

→ 未来方向：Reaching阶段用EEF delta，接触阶段切换Joint

三篇论文，同一套动作语言

它们为什么都不约而同选择了「以当前 EE 为锚点的相对增量」？
因为这套表示同时解决了三个核心工程问题。

UMI
2024 · Columbia / Toyota
relative EE trajectory

所有未来 pose 都相对于当前同一个 EE pose，而非逐帧递推。整段 trajectory 共享一个 anchor，模型只需学一个紧凑的局部动作分布。

单锚点避免逐帧递推的误差累积；动作空间均值接近零、方差小，显著降低学习难度。

GEAR-VLA
2026 · 跨本体迁移框架
Relative End-effector Action

明确强调不是相对前一预测帧，而是始终锚定当前观测帧的 EE pose。输出端保持本体无关，输入端保留关节状态作为上下文。

锚定观测帧切断自回归误差链；EE 层面本体无关，同一策略可直接迁移到不同机器人构型。

GR00T N1.7
NVIDIA · 人形机器人基座模型
Relative EEF Action Space

未来动作定义为相对于 current pose 的 delta，贯穿整个预测 horizon。一个 anchor 多步输出，天然适配 chunk prediction 范式。

相对表示消除绝对坐标系下的分布偏移；chunk 输出减少推理次数，提升实时控制吞吐效率。

三种叫法，同一本质：以当前帧 EE 为锚点，预测相对增量。

IK 的角色

但 IK 算法仍然是必需的

场景一：脚本采集

在脚本采集的时候通常都是设定对应的 EEF waypoint，然后通过 IK 算出对应的 joints，而不是直接写死关节角度。

场景二：数据扩充（如 MimicGen）

在类 mimicGen 的数据扩充功能中，算法本身就是基于物体的位姿 EEF 相对变化，此时更需要一个好的 IK 算法来对 joint 进行约束，避免奇异值等多种问题，从而得到更加平滑可用的轨迹。

（这里不细说具体每一个 IK 算法的特征，毕竟经典的有 PINK IK、Lula IK 等）

如何选择 IK？

研究者通常不会临时挑一个所谓排行榜第一的 IK，而是优先沿用：

  • →采集数据时使用的控制器
  • →机器人厂商已经验证过的笛卡尔控制接口
  • →对应 benchmark 环境的标准控制器

→ 降低训练和部署之间的控制分布差异

MimicGen pipeline：基于 EEF 相对变化的数据扩充

IKSel：种子选择对 IK 求解质量的影响

IK 小结

三个核心认知

一、IK 是 EEF 到执行的必经桥梁

模型输出 EEF 后，必须经 IK 求解才能得到可执行的关节指令。IK 算法质量决定最终执行效果。

二、IK 在数据侧仍然不可或缺

脚本采集依赖 IK 从 EEF waypoint 反算关节角；MimicGen 通过 EEF 相对变换生成新轨迹，仍需 IK 反算平滑关节序列；仿真数据生成（如 Isaac Sim）同样依赖 IK 将 EEF 规划转为关节轨迹。

三、IK 算法选择与求解速度

IK 多解歧义需要合理的种子选择策略（如 IKSel）；高频回路要求 IK 足够快，关键是快且不牺牲解的质量。

总之，IK 不仅决定模型输出的最终执行质量，也是数据采集与扩充（如 MimicGen、Isaac Sim）流水线的关键一环 —— 没有好的 IK，再好的 EEF 预测也落不了地。选对 IK 算法、保证求解又快又稳，是让 EEF 方案真正可用的最后一块拼图。

论文支撑 3 + 新范式

IKSel + 新训练范式

arxiv.org/html/2503.22234v1

新范式核心

VLA 输入 EEF → 输出 EEF → IKSel 选择最优种子 → 数值IK快速求解 Joint

IKSel 核心思路

  • →为快速数值IK迭代选择合适的种子关节值
  • →从多个候选种子中评估选择最优起点
  • →确保IK收敛快、解连续、避免奇异点
  • →结合 Joint 参考列表避免极端构型

完整Pipeline

  • →Step 1: VLA 模型从视觉输入预测 EEF 目标（本体无关）
  • →Step 2: IKSel 根据当前 Joint 列表选择最优 IK 种子
  • →Step 3: 数值 IK 从最优种子快速收敛到目标关节角
  • →Step 4: 输出 Joint Command 执行
